Imporant FAQ

1. What is the Document Object Model (DOM)?

The Document Object Model (DOM) is a programming interface (API) for HTML and XML documents. It represents the page as a hierarchical tree structure where every element, attribute, and piece of text is a node. JavaScript uses the DOM to access and manipulate the structure, style, and content of a web page, enabling dynamic updates and interactivity.

2. How do you select an element from the DOM?

i. document.getElementById('idName'): Selects a single element by its unique id.

ii. document.querySelector('.className'): Selects the first element that matches a specific CSS selector (e.g., class, ID, or tag name).

iii. document.querySelectorAll('tagName'): Selects all elements that match a CSS selector, returning a NodeList (array-like object).

3. What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique where you attach a single event listener to a common parent element rather than attaching individual listeners to multiple child elements.

It is useful because:

i. Efficiency: It saves memory and processing power by requiring only one listener, reducing overhead.

ii. Dynamic Elements: It automatically handles events for elements that are added to the DOM after the page loads without needing to re-attach listeners.

4. How do you manipulate an element's attributes and styles using the DOM?

Attributes are manipulated using methods on the element node:

i. element.setAttribute('attr', 'value'): Adds or changes the value of an attribute (e.g., src, href).

ii. element.getAttribute('attr'): Retrieves the current value of an attribute.

iii. element.removeAttribute('attr'): Removes an attribute.

Styles are manipulated via the style property:

i. element.style.backgroundColor = 'red': Sets inline CSS properties using camelCase (e.g., background-color becomes backgroundColor).

ii. element.classList.add('new-class'): The recommended way: adds, removes, or toggles CSS classes defined in a stylesheet.
